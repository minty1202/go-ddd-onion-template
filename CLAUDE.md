# Todo App

## 常時適用ルール

@.claude/rules/advice.md
@.claude/rules/no-code.md

## 前提

- このプロジェクトは今後の Go 新規開発の土台（テンプレート）にする前提で作っている
- 「DDD + オニオンのテンプレートとして実践的に一番良い構成はどれか」を研究しながら構築する
- 研究・議論で得られた設計上の気づきは `.claude/rules/` にルールとして残す（その場の結論で終わらせない）

## 目的

- Go の実践的なプロジェクトを Todo アプリを作りながら学ぶ
- DDD + オニオンアーキテクチャを学ぶ
- Todo 固有のアドバイスではなく、Go プロジェクト全般に通用するアドバイスをすること
- アドバイスが実践的なプロジェクトで通用するかわからないときは、推測せず必ず調べること

## 進め方

- 設計判断や選択肢の提示は、比較表や候補を一気に並べず、**対話的に 1 論点ずつ**確認しながら進める
- 調査結果が広いときも、要点 1〜2 行と「どこを深堀りするか」の問いで返す。網羅的な列挙はユーザーが明示的に求めたときだけ
- 議論が発展した場合は、**まず Go 標準ライブラリがどう扱っているか** を確認してから選択肢を検討する
