# Audit Rules

`.claude/rules/` 配下（**サブディレクトリ含む全 Markdown ファイル**）のルール文書品質を `rules-auditor` エージェントで監査する。

## 手順

1. `Agent` ツールで `subagent_type: rules-auditor` を起動
2. プロンプトには次の 2 点を必ず含める（評価観点は `rules-auditor` 側に定義済みなので body では繰り返さない）:
   - 「**最初に `Glob` ツールで `.claude/rules/**/*.md` を再帰的に列挙し、列挙結果を報告冒頭に明示してから**、全ファイルを `Read` して評価観点に従って findings を報告してください」
   - 「`Read` のパス推測でファイルを集めるのは禁止。必ず `Glob` の列挙結果を起点にすること」
3. 返ってきた findings の冒頭でファイル列挙結果（カバレッジ）を確認し、`.claude/rules/` 配下の Markdown が全て監査対象になっているかを検証する
4. 返ってきた `要再考` と `矛盾` をユーザーに対話的に提示し、修正方針を確認する

## 注意

- ルール文書の**品質**を評価するスキル。コードのルール準拠チェックには使わない
- 監査結果はユーザーの判断で取捨選択する（エージェントは中立評価）
